Confluence parsing:

1. Fetch the page body in Confluence storage format.
2. Parse all table rows.
3. Identify rows whose status/open-state cell indicates the row is still open. Treat normalized cell values such as `open`, `todo`, `not done`, `failing`, `failed`, or `needs work` as open only when they appear in a status-like column or in the row's explicit status cell. Do not treat a prose sentence that merely contains the word "open" as an open status.
4. For every open row, preserve every source cell value in source order. Also capture the table index, row index within the table, normalized status, and the feedback/problem cell text.
5. Extract any chat id from the row feedback/problem text. Recognize explicit forms such as `chatid=abc123`, `chatId abc123`, `chat id: abc123`, or a full Observability AI Chat URL with a `chatid` query parameter. Current browser chat IDs are usually six alphanumeric characters, but database `ConversationId` can also be a UUID. Do not infer a chat id from unrelated six-character strings.

Conversation query:

Use the extracted `chatid` as `ConversationId`.

```sql
SELECT
  ObservabilityChatTurnId,
  TurnId,
  ConversationId,
  ConversationTitle,
  RunId,
  PromptHash,
  UserEmail,
  UserMessage,
  AssistantReply,
  ToolCallSummaryJson,
  IsSuccess,
  ErrorMessage,
  WhenCreated
FROM marioai.ObservabilityChatTurn
WHERE ConversationId = @chatid
ORDER BY WhenCreated, ObservabilityChatTurnId;
```

Optional feedback query:

```sql
SELECT
  ObservabilityChatMessageFeedbackId,
  ConversationId,
  MessageIndex,
  MessageContentHash,
  FeedbackType,
  CommentText,
  UserEmail,
  WhenCreated
FROM marioai.ObservabilityChatMessageFeedback
WHERE ConversationId = @chatid
ORDER BY WhenCreated, ObservabilityChatMessageFeedbackId;
```

Optional state snapshot query:

```sql
SELECT
  ObservabilityChatStateSnapshotId,
  ConversationId,
  TurnId,
  MessageRole,
  MessageIndex,
  MessageContentHash,
  StateMachineVersion,
  Status,
  ActiveStateId,
  ActiveStateLabel,
  EvaluationCount,
  YesEvaluationCount,
  EvaluationsJson,
  ResolverJson,
  ErrorMessage,
  WhenCreated,
  WhenUpdated
FROM marioai.ObservabilityChatStateSnapshot
WHERE ConversationId = @chatid
ORDER BY MessageIndex, MessageRole, WhenCreated, ObservabilityChatStateSnapshotId;
```

Full conversation formatting:

- Preserve all turns in chronological order.
- For each row from `ObservabilityChatTurn`, emit:
  - `USER` message with `WhenCreated`, `TurnId`, and `UserEmail`
  - `ASSISTANT` message with `AssistantReply`, `ToolCallSummaryJson`, `IsSuccess`, and `ErrorMessage` when present
- Include linked message feedback and state snapshot summaries after the relevant assistant message when available.
- If the row has a chatid but no database rows are found, set the full conversation cell to `CHATID FOUND BUT NO DATABASE CONVERSATION ROWS FOUND`.
- If the row has no chatid, leave the full conversation cell blank and set the chatid cell to blank.
